Layer 1
Feature Files (BDD)
.feature files describe expected behavior in natural language (Gherkin). They form the contract between business and technical teams, written in Given / When / Then format.
# login.feature
Feature: User Authentication
 
  Scenario: Successful login
    Given "the user is on the login page"
    When  "they enter valid credentials"
    Then  "they should see the dashboard"
// 05 — Bug Hunter
Documented bugs
Found on public QA demo applications — professional format: steps to reproduce, expected vs actual, severity.
Negative quantity accepted at checkout
High
🔧 App: OpenCart Demo 📁 Type: Functional
The cart quantity field accepts negative values. The displayed total becomes negative and the order goes through without any server-side validation.
Steps to reproduce
  1. Add a product to cart
  2. Modify quantity to -1 in the field
  3. Click "Update" — total becomes negative
  4. Proceed to checkout — order is accepted
Expected: "invalid quantity" error message
Actual: negative total, order validated without error
Validation · Business Logic opencart-demo.com
Dashboard access without authentication
Critical
🔧 App: SauceDemo 📁 Type: Security
Navigating directly to /inventory.html without an active session loads the product page partially without any redirect to login.
Steps to reproduce
  1. Open browser without active session
  2. Navigate directly to /inventory.html
  3. Observe — page loads without login
  4. Product data is visible
Expected: automatic redirect to /login
Actual: partial access to protected content
Security · Authorization saucedemo.com
Product sort by price — alphabetical instead of numeric
Medium
🔧 App: SauceDemo 📁 Type: Functional
The "Price low to high" sort compares prices as strings instead of numbers. Result: $10 appears before $7.
Steps to reproduce
  1. Login with valid account
  2. Select "Price (low to high)" in filter
  3. Observe displayed product order
Expected: $7.99 → $9.99 → $15.99 → $29.99
Actual: alphabetical order — $15.99 before $7.99
Functional · Sorting saucedemo.com
Add to cart without session — false confirmation
High
🔧 App: Demoblaze 📁 Type: UX / Functional
An unauthenticated user receives "Product added" confirmation after adding to cart, but the item is not stored anywhere. No message prompts to login.
Steps to reproduce
  1. Access Demoblaze without login
  2. Click on product then "Add to cart"
  3. Alert confirms "Product added"
  4. Login and check cart — empty
Expected: login redirect or explicit message
Actual: false confirmation, data silently lost
UX · State Management demoblaze.com
