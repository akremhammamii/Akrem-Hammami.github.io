.feature files describe expected behavior in natural language (Gherkin). They form the contract between business and technical teams, written in Given / When / Then format.
# login.feature
Feature: User Authentication
 
  Scenario: Successful login
    Given "the user is on the login page"
    When "they enter valid credentials"
    Then "they should see the dashboard"

Documented bugs

Negative quantity accepted at checkout
High
App: OpenCart Demo Type: Functional
Cart quantity field accepts negative values. Total becomes negative and order proceeds without server-side validation.
Steps to reproduce
  1. Add any product to cart
  2. Change quantity to -1 in the field
  3. Click "Update" total becomes negative
  4. Proceed to checkout order is accepted
Expected: validation error "quantity must be greater than 0"
Actual: negative total, order validated without error
Validation Business Logicopencart-demo.com
Dashboard access without authentication
Critical
App: SauceDemo Type: Security
Navigating directly to /inventory.html without an active session loads the products page partially without any redirect to login.
Steps to reproduce
  1. Open browser without active session
  2. Navigate directly to /inventory.html
  3. Observe page loads without authentication
  4. Product data is partially visible
Expected: automatic redirect to /login
Actual: partial access to protected content

Price sort alphabetical instead of numeric
Medium
App: SauceDemo Type: Functional
"Price low to high" sort compares prices as strings rather than numbers. Result: $10.99 appears before $7.99.

  2. Select "Price (low to high)" filter
  3. Observe the product order displayed
